ForwardPartitioner

DataStream => DataStream

这么一看好像 GlobalPartitioner 和 ForwardPartitioner 没什么不一样，但存在即合理，因为ForwardPartitioner 有个重要的前提条件就是上下游的并发度必须一样，如果不一样是会抛异常的。由示意图也能看出来，GlobalPartitioner 是将上游所有算子的数据发给下游一个算子，而 ForwardPartitioner 则是上游的算子和下游的算子是 1:1 对应的。

RescalePartitioner

DataStream => DataStream

Rescale 可以翻译为缩放，顾名思义，RescalePartitioner 可以通过缩放处理上下游算子并发度不一样的情况。

例如，上游并发度为 2 分别为 A、B，下游并发度为 4 分别为 1、2、3、4，RescalePartitioner 会将 A 的数据发给 1、2，将 B 的数据发给 3、4；再比如上游并发度为 4 分别为 A、B、C、D，下游并非度为 2 分别为 1、2，RescalePartitioner 会将 A、B 的数据发给 1，将 C、D 的数据发给 2。

使用方式，作用于 DataStream => DataStream 直接 .rescale

RebalancePartitioner

DataStream => DataStream

RebalancePartitioner 会循环发送给下游每一个实例

使用方式，作用于 DataStream => DataStream 直接 .rebalance

ShufflePartitioner

DataStream => DataStream

使用方式，作用于 DataStream => DataStream 直接 .shuffle

BroadcastPartitioner

DataStream => DataStream

BroadcastPartitioner 会将上游实例的数据发给下游每一个实例，相当于广播

使用方式，作用于 DataStream => DataStream 直接 .broadcast

KeyGroupStreamPartitioner 是根据 key 取 hash 分组选择下游 channel

使用方式，直接 .keyBy，并传入 key selector
